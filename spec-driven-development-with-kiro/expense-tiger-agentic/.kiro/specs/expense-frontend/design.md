# Design: Expense Frontend (Vanilla JS SPA)

## Context

See `.kiro/specs/expense-submission-app/design.md` for full architecture. This spec covers only the frontend SPA implementation.

## Frontend Components

### Router (`frontend/js/router.js`)
- Hash-based routing: #/auth, #/upload, #/edit, #/expenses
- Listen to hashchange events

### AuthModule (`frontend/js/auth.js`)
- Cognito Identity SDK (via CDN)
- signUp, confirmSignUp, signIn, signOut
- SRP authentication, session management
- Auth gate rendering

### ReceiptUploader (`frontend/js/receipt-uploader.js`)
- File input + drop zone
- Validate type (JPEG/PNG/WebP) and size (≤5MB)
- Image preview via FileReader
- Base64 encode and POST to /expenses/extract

### ExpenseEditor (`frontend/js/expense-editor.js`)
- Editable form for all extracted fields
- Line items with add/remove
- Total recalculation
- Required field validation

### ExpenseList (`frontend/js/expense-list.js`)
- GET /expenses, render cards sorted by date desc
- Loading and empty states

### API Client (`frontend/js/api.js`)
- snake_case ↔ camelCase conversion
- JWT header injection
- 401 → redirect to auth gate

## Frontend config
- `frontend/config.js` generated by deploy script with apiEndpoint, userPoolId, userPoolClientId, region

## Correctness Properties (Frontend)
- Property 1: Password policy validation
- Property 2: File upload validation
- Property 4: Line item total calculation
- Property 5: Expense submission validation
